# Sprint 2 Goal

**Sprint Duration**: March 30 - April 26, 2026 (4 weeks)

---

## 🎯 Sprint Goal

**"Capstone-ready RAG with confidence gating and Documentation Gap Radar differentiator"**

---

## Success Criteria

By end of Sprint 2, we will have:

1. ✅ **Confidence Scoring & Gating** - Detect low-evidence answers and provide safe fallback responses
2. ✅ **Documentation Gap Radar** ⭐ - Auto-log low-confidence questions to database with frequency tracking and dashboard UI
3. ✅ **Sources Page** - Show indexed documents, sync status, and manual re-sync capability
4. ✅ **Evaluation** - First evaluation run (5-10 test questions) to validate system quality
5. ✅ **Sprint Review & Retrospective** - Document learnings and prepare for Sprint 3

---

## Why This Matters

**Confidence gating** prevents our system from giving unsupported answers (production safety).

**Gap Radar** is our **capstone differentiator** - it goes "above and beyond" basic RAG to demonstrate initiative and production thinking. This feature targets **4-5/5 rubric scores** vs. 3/5 for basic implementations.

**Sources page** provides operational transparency and allows users to understand what documentation is indexed.

---

## Feature Freeze

After Sprint 2 review (Apr 26), **no new user-facing features** unless required to fix a blocker for MVP acceptance criteria. Sprint 3 focuses on testing, deployment, and demo preparation only.

---

## Definition of Done

- All critical path tasks complete (confidence, Gap Radar, sources)
- Gap detection working end-to-end (question → log → dashboard)
- Evaluation run completed with documented results
- Sprint retrospective written
- Demo-ready for Sprint 2 features
